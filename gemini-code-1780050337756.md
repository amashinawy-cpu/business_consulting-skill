# Presentation Storylining & Architecture Workflow

## Step 1: Deck Scoping & Audience Calibration
- Define the core objective of the meeting (e.g., Approval for a Holding Company model, funding for an ICT training lab, green-lighting a sustainable tech integration).
- Identify the strict target audience (e.g., Board of Directors, Ministry Officials, or Venture Capital Investors).
- Establish the target slide count budget (e.g., 5-slide elevator pitch, 10-slide executive summary, or 20-slide deep dive).

## Step 2: Narrative Arc Mapping (The Skeleton)
- Draft a chronological list of slide titles to ensure the logical flow holds up without reading the content.
- Use a classic consulting arc: *The Hook/Current Challenge $\rightarrow$ The Core Solution $\rightarrow$ Operational Mechanics $\rightarrow$ Financial/Strategic ROI $\rightarrow$ The Ask/Implementation Roadmap.*

## Step 3: Slide-by-Slide Visual Construction
- Execute the template defined in the system prompt for every slide in the skeleton.
- Alternate visual layouts between consecutive slides to keep the audience visually engaged (e.g., follow a 3-column matrix with a horizontal timeline, never use back-to-back heavy text tables).
- Ensure all business metrics, percentages, or milestones stand out as prominent visual callouts.

## Step 4: Final Message Review & Refinement
- Audit the deck to strip away passive language, fluff, or dense text blocks.
- Ensure the presenter's verbal notes match the visual layout seamlessly.
- Export the complete structural blueprint in clean, copy-pasteable Markdown format.
